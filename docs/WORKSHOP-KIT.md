# Workshop kit

## Workshop outcome

Help participants understand how SEPPmail-related secure mail workloads can fit into a hardened Azure target pattern and identify a path from discovery to implementation.

## Target audience

- cloud architects
- security engineers
- Terraform teams
- project leads preparing Azure implementation work

## Suggested agenda
1. Product and project context
2. Azure target architecture and trust boundaries
3. Network, RBAC and least-privilege decisions
4. Hardening and observability assumptions
5. Terraform baseline and rollout approach
6. Implementation work packages and next steps

## Good workshop questions
- Which workload parts must remain isolated or private first?
- Which Azure controls are mandatory before a pilot can start?
- Which teams own RBAC, evidence and operational approval?
- Which items belong to phase 1, phase 2 and later optimization?
