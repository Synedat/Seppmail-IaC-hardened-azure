# Demo walkthrough for Seppmail-IaC-hardened-azure

## Objective
Show how **Seppmail-IaC-hardened-azure** can be introduced in a customer conversation, workshop or engineering handover.

## Suggested flow
1. Open the repository overview and explain the scope: Terraform starter for hardened Azure deployments supporting secure mail workloads.
2. Show the quick start and one working example from `examples/` or the repo root.
3. Use the files in `demo-data/` to present realistic output without needing a live target system.
4. Walk through architecture, permissions, security/compliance mapping and operations guidance.
5. Close with extension ideas and the Synedat contact path.

## Demo assets to show
- `demo-data/sample-result.json`
- `demo-data/sample-config.yaml`
- `demo-data/sample-health-report.csv`
- `docs/images/landing-page-wireframe.svg`
- `docs/images/homepage-hero.svg`

"This repository is intentionally public-facing, implementation-oriented and easy to adapt. It helps teams accelerate architecture discussions, workshops and controlled onboarding in the SEPPmail ecosystem."
